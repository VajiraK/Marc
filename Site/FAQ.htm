1 What is MARC?
MARC (Millennium All-Island Robot Competition) is part of Millennium IT’s 10th year celebration. It is a programming competition aimed at promoting the software industry among school children and university students in an exciting and easy way.

You must program a robot to safely navigate a virtual environment, pickup packages and return to a starting position. The scoring is on time, safety, number of packages picked up and their weight and volume.
2 What are the prerequisites that I need to have to install MARC?
  • Windows XP
  • Windows 2000
  • Windows NT
The following compilers will be supported for the relevant programming languages:

Language Compiler
C, C++ Microsoft Visual C++ 6, 2003 or 2005
Java JDK 1.1 or above
VB Microsoft Visual Basic 6
.NET Microsoft .NET framework 1.x
3 What is MARC SDK?
The SDK (Software Development Kit) contains applications that will allow you to create test maps and a virtual environment to play the game and test the robots behavior real time. To download the MARC SDK log on to www.marc.lk
4 How do I enter the competition?
In order to compete in the MARC you must be a registered user. Register now.
5 What is tested in this competition?
Problem analysis, designing and programming skills.
6 Do I need be an expert developer (programmer) to participate in this competition?
Not at all. If you think you have the ability to program the robot according to the competition rules then you can participate.
7 What programming languages can be used to program the robot?
You may use C, C++, VB (Visual Basic), Java, or a .NET language.
8 How do I win this competition?
Scores are calculated based on the number packages retrieved by the robot. The highest scorer will win the competition.
9 What is the length of a door?
The doors in the Spaceship are one block thick and can vary in width from one block to any size in multiples of block. However the robot can only pass through doors that are at least 5-blocks wide.
10 Do I have to start the Spaceship program (the virtual environment) from my program?
No. Write you program assuming that the Spaceship application is already started. You only have to worry about the behavior of the robot.
11 How does the robot perceive its coordinate system?
The robot's vision consists of a 5x5 matrix. The top-left position represents the origin (0,0).
12 Is the size of the map fixed?
No. The size varies from map to map.
13 To open a door, in which position does the robot need to be positioned with respect to the door?
The robot and the door should be left aligned. In other words, the robot needs to be at the leftmost corner looking at the door.
14 What is 'teleporting'?
Some areas of the map contain teleporting beams. When the robot enters one of these beams it can move to a different location in the spaceship (or the map) without utilizing battery.
15 Will you provide sample robots?
Yes. A sample program for each programming language will be provided in the SDK.
16 What happens to the opened doors?
Once opened, a door remains open.
17 How do I submit my program (robot)?
Submit you program by logging on to the MARC website. There will be a link in the home page (after you log in). You must first register yourself in the competition to upload the program.
18 What is MD5 and why am I given one each time I upload a file?
The 32 letter MD5 string (in hexadecimal) is what's known as a hash. It can help you to validate your submission. The MD5 hash is calculated soon after the robot upload. The command line program on this website can be used to check the MD5 hash of your robot. You need to do this before uploading the robot to our server and then compare it against the hash generated by the server (what's shown on the confirmation page).
19 I can't remember my password. How can I login?
We can reset your password to a random string of characters and send it to the email address you specified at the time of registration. Go to the Help page to reset your password.
20 Is there a limit to the number of submissions?
No; there are no limits. However we only keep the most recent submission of a robot; in other words, a new submission will replace your previous submission. Note that the most recent submission is taken for the batch run.
21 What happens when a batch run is postponed?
Batch runs are scheduled to take place weekly. But in case of a postponement the batch run will take place the following week and all robots submitted up to this date will be evaluated.
